Tim Berners-Lee and the World Wide Web

The World Wide Web was born from Tim Berners-Lee's idea at CERN to develop a single information system that would create a platform for document sharing among professors, students and all academics.

In 1989, while working at CERN (European Organization for Nuclear Research), Berners-Lee proposed a global hypertext project that would eventually become the World Wide Web. His vision was to create a system where physicists could share information across different computer systems.

The First Web Browser

In 1990, Berners-Lee developed the first web browser-editor called WorldWideWeb (later renamed Nexus) and the first web server. This allowed researchers to view and edit hypertext documents, creating the foundation for the web as we know it today.

HTML Development

HTML, a basic markup language based on SGML (Standard Generalized Markup Language), was first presented by Berners-Lee in 1991. With only 18 elements, this early iteration of HTML was mostly concerned with document structure, including heading, paragraph, and simple linking tags.

Evolution of HTML

Version Year Key Features
HTML 1.0 1991 Original 18 elements focused on document structure
HTML 2.0 1995 Forms, tables, internationalization features
HTML 3.2 1997 Support for scripting, enhanced tables, text flow around images
HTML 4.01 1999 Cascading Style Sheets, scripting improvements, accessibility features
XHTML 1.0 2000 Reformulation of HTML as an XML application
HTML5 2014 Native audio/video support, canvas for drawing, local storage, semantic elements

HTTP and Web Communication

Hypertext Transfer Protocol (HTTP) serves as the main primary protocol for web communication, allowing for communication between servers and browsers by sending get requests and receiving HTML text of the required page. The presentational and structural guidelines that browsers use to render web pages are provided by this HTML content.

How HTTP Works

  1. A user enters a URL in their browser or clicks a link
  2. The browser sends an HTTP request to the server
  3. The server processes the request and retrieves the necessary files
  4. The server sends an HTTP response containing the requested HTML
  5. The browser renders the HTML content into a visual web page

This client-server model became the foundation of web communication and remains the basis of how we interact with web content today.
